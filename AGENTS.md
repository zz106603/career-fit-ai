1. 모든 응답은 한국어로 작성한다.
2. 작업 전 GitHub Issue, docs 최신본, README, 관련 설정을 먼저 확인한다.
3. 모르는 구조는 추정하지 않고, Issue 범위를 벗어난 수정이나 과도한 설계는 하지 않는다.
4. Spring Boot 코드는 반드시 backend/ 아래에서 작업한다.
5. 사용자 경력에 없는 정보를 생성하지 않고, 미확정 AI 후보는 분석 근거로 사용하지 않는다.
6. 근거가 없으면 확인 불가로 처리하고, 미충족에는 명시적 충돌 근거가 필요하다.
7. 회사 조사 실패가 경력 매칭 전체 실패로 이어지지 않게 한다.
8. 내부 Multi-Agent와 MCP를 사용하지 않으며, 사용자 데이터와 Vector 검색에는 userId 범위를 적용한다.
9. 작업 완료 후 변경 내용, 이유, 테스트 결과, 남은 리스크를 요약한다.
